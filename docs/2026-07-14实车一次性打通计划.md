`install_release.sh` 会切换 `current`、启动服务、执行健康检查；失败会自动恢复旧 `current`。不要在它运行时手工重启容器。

```bash

sudo systemctl edit icar_startup.service
```

`/etc/icar/icar.env` 只在现场填写实际值：

```text
ICAR_MQTT_HOST=<broker>

ICAR_MQTT_PASS=<password>

ICAR_DEVICE_ID=<robot-id-or-empty>

DEEPSEEK_API_KEY=<optional>
```

systemd drop-in 内容：

```ini
[Service]
EnvironmentFile=-/etc/icar/icar.env
```

然后执行：

```bash

journalctl -u icar_startup.service -n 200 --no-pager
```

验收：安装脚本成功退出，`current` 指向 `$VERSION`，两个 systemd 服务为 active，`/health` 返回 HTTP 200。

## 十六、现场故障分流

| 现象 | 首查 | 处理方向 |
|---|---|---|
| 启动服务失败 | `journalctl -u icar_startup -n 200` | 找到第一个失败阶段，不重复盲目重启 |
| ROS 节点互相看不到 | 两容器 `ROS_DOMAIN_ID` | 统一为 30，重启 daemon/节点 |
| 底盘不动 | `/dev/myserial`、driver、`/cmd_vel` | 先查串口，再查 mux，不绕过安全链 |
| 松手仍运动 | `/cmd_vel_cloud`、租约、mux | 立即物理断电，判 P0 失败 |
| 视频 503 | `/tmp/icar_camera.log`、`/tmp/icar_mjpeg.log` | 查相机占用、Topic 是否有帧 |
| 本地视频好、远程截图失败 | `/tmp/cloud_bridge.log`、6502 | 查 snapshot Topic、loopback JPEG、大小限制 |
| MQTT 离线 | `/tmp/cloud_bridge.log`、Broker TCP | 查网络、参数、ACL、device_id |
| APP 浏览器能开但 APP 不通 | Android 权限和 VPN | 关 VPN/移动路由切换，查看 logcat |
| Nav2 不启动 | map/params/localization | 保持 `ICAR_ENABLE_NAV2=0`，先修导航前置 |
| 新版本异常 | `current/previous`、systemd | 立即运行回滚脚本 |

常用日志：

```bash
journalctl -u icar_startup.service -n 300 --no-pager
journalctl -u icar_web_gateway.service -n 200 --no-pager
docker exec icar_ros2 tail -n 100 /tmp/cloud_bridge.log
docker exec icar_ros2 tail -n 100 /tmp/icar_vision.log
docker exec icar_ros2 tail -n 100 /tmp/icar_mjpeg.log
docker exec autodrive_ros2 tail -n 100 /tmp/velocity_mux.log
docker exec autodrive_ros2 tail -n 100 /tmp/app_bridge.log
```
