## 2. 模型与技术选型

### 2.1 推荐方案

中期采用“三层兜底”方案：

1. **基础可演示层：OpenCV/颜色规则检测**
   - 已在 `vision_node` 中实现。
   - 用于无模型、无网络、车端环境不稳定时保底。
   - 可检测仿真中的红色路障、绿色标志物、黄色行人目标。

2. **成熟模型层：YOLO 预训练模型**
   - 车端能安装/已有 `ultralytics` 时，直接加载 YOLO 模型。
   - 中期建议先识别 `person`、`bottle`、`chair`、`backpack` 等 COCO 常见类。
   - 模型文件优先选轻量模型：`yolo11n.pt`、`yolo26n.pt` 或老师手册里的 `yolov5s.pt`。

3. **加分拓展层：自定义路障/积水模型**
   - 使用去年已有路障数据集或上车采集少量图片。
   - 类别先控制在 1-2 类：`obstacle`、`water`。
   - 中期只需要说明路线并展示少量标注/采集样例；最终前再训练/微调。

### 2.2 为什么不主攻交通标志

老师原始课程要求中，加分项更明确的是“道面缺陷、湿滑积水、道面异常目标检测”等；交通标识识别主要出现在视觉实验手册里，属于 YOLO 应用案例。你们的小车在教室/走廊短距离巡检，交通标志未必自然出现，且公开数据多为车载/监控视角，和小车低机位不完全匹配。

所以视觉主线建议表述为：

> 完成深度相机接入、图像采集、人员/普通障碍物/积水或湿滑区域检测和结果发布；交通标志识别作为实验手册参考，不作为中期主线；摔倒老人不作为视觉模块主识别目标，后续按雷达/地图建模或系统融合异常处理。

### 2.3 当前代码支持

当前 `vision_node` 支持两种检测后端：

| 后端 | 参数 | 说明 |
|---|---|---|
| `auto` | 默认 | 有 YOLO 模型就用 YOLO，否则自动回退 OpenCV 轻量检测 |
| `yolo` | 需要 `YOLO_MODEL` | 强调使用 YOLO；如果模型加载失败会提示并回退 |
| `color` | 不需要模型 | 只用轻量检测，适合中期无车/无模型演示 |

输出统一发布到：

```text
/vision/detections
```

主接口消息是 `icar_interfaces/DetectionArray`，包含类别、置信度、bbox、截图路径等字段，方便 APP/任务调度读取。节点同时保留 `/vision/detections_json` 调试输出，方便临时查看检测后端、模型名等扩展信息。

## 3. 无车时怎么跑

在 Ubuntu 20.04 + ROS2 Foxy 环境中：

```bash
cd ~/ros2_ws
colcon build --symlink-install --packages-select vision_patrol
source install/setup.bash
```

终端 1：启动仿真相机。

```bash
ros2 run vision_patrol fake_camera --ros-args \
  -p image_topic:=/camera/color/image_raw
```

终端 2：检查相机帧。

```bash
ros2 run vision_patrol camera_probe --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_first_frame:=true
```

终端 3：启动检测节点，先用保底方案。

```bash
ros2 run vision_patrol vision_node --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p detector_backend:=color \
  -p publish_annotated:=true \
  -p enable_road_detection:=true
```

终端 4：查看检测结果。

```bash
ros2 topic echo /vision/detections
ros2 topic echo /vision/detections_json
ros2 topic echo /vision/annotated_image
```

可截图内容：

- `fake_camera` 正在发布。
- `camera_probe` 收到帧并保存首帧。
- `/vision/detections` 有 `obstacle/water/sign/person` 检测输出。
- `/vision/annotated_image` topic 存在。

## 4. 上车时怎么跑

### 4.1 进入正确容器

宿主机：

```bash
source ~/.bashrc
is
id
```

进入 `icar_ros2` 容器后：

```bash
source /opt/ros/foxy/setup.bash
m1
ros2 topic list | grep -E "image|camera|depth|rgb|color|astra|points"
```

重点记录真实 RGB topic。常见候选：

```text
/camera/color/image_raw
```

检查频率：

```bash
ros2 topic hz /camera/color/image_raw
```

如果没有频率，先查相机是否被占用：

```bash
ps -ef | grep -E "astra|camera|colorHSV|colorTracker"
```

### 4.2 构建视觉包

把本仓库的 `vision/vision_patrol` 放到车端 ROS2 工作空间，例如：

```text
~/ros2_ws/src/vision_patrol
```

然后：

```bash
cd ~/ros2_ws
colcon build --symlink-install --packages-select vision_patrol
source install/setup.bash
```

### 4.3 先跑相机探测

```bash
ros2 run vision_patrol camera_probe --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_first_frame:=true
```

通过后，把保存图片路径截图或记录下来。

### 4.4 跑保底检测

```bash
ros2 run vision_patrol vision_node --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p detector_backend:=color \
  -p publish_annotated:=true
```

查看：

```bash
ros2 topic echo /vision/detections
ros2 topic list | grep vision
```

### 4.5 跑 YOLO 检测

前提：车端 Python 环境中已经有 `ultralytics`，并且模型文件在车上。

建议模型路径：

```text
/home/jetson/models/yolo11n.pt
/home/jetson/models/yolo26n.pt
/home/jetson/models/yolov5s.pt
```

启动：

```bash
ros2 run vision_patrol vision_node --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p detector_backend:=yolo \
  -p yolo_model:=/home/jetson/models/yolo11n.pt \
  -p yolo_device:=0 \
  -p yolo_confidence:=0.35 \
  -p yolo_imgsz:=640 \
  -p target_classes:="[person,bottle,chair,backpack]" \
  -p publish_annotated:=true
```

如果 `yolo11n.pt` 不兼容，优先改成老师手册一致的：

```bash
-p yolo_model:=/home/jetson/models/yolov5s.pt
```

中期不要把时间耗在安装最新版环境上；能稳定输出检测结果更重要。

## 5. 数据集/截图采集怎么做

启动采集节点：

```bash
ros2 run vision_patrol dataset_recorder --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_dir:=/tmp/icar_vision_dataset \
  -p max_images:=200
```

单张截图：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"capture_once\",\"tag\":\"midterm_sample\"}'}"
```

每 3 秒采一张，用于路障/积水小数据集：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"set_interval\",\"interval_sec\":3.0}'}"
```

停止：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"stop\"}'}"
```

建议采集清单：

| 类别 | 数量 | 说明 |
|---|---:|---|
| `obstacle` | 50-100 | 箱子、书包、纸箱、锥桶，从远/中/近、左/中/右拍 |
| `water` | 50-100 | 积水、水渍、反光湿滑区域，注意不同光照和地面材质 |
| `person` | 不强制 | 预训练模型已有，现场只做检测验证 |

## 6. 目标跟踪怎么演示

先启动检测节点，再启动跟踪节点：

```bash
ros2 run vision_patrol target_tracker --ros-args \
  -p detections_topic:=/vision/detections \
  -p cmd_vel_topic:=/vision/target_cmd_vel \
  -p enabled_on_start:=false \
  -p target_classes:="[person]"
```

开始追踪：

```bash
ros2 topic pub --once /vision/target_tracking/command std_msgs/String \
  "{data: '{\"action\":\"start\",\"class_name\":\"person\"}'}"
```

观察速度建议：

```bash
ros2 topic echo /vision/target_cmd_vel
ros2 topic echo /vision/target_tracking/status
```

安全要求：

- 中期只展示 `/vision/target_cmd_vel`，不要直接转发 `/cmd_vel`。
- 真实小车运动由组内任务调度/底盘控制统一决定，避免和雷达导航抢控制权。

## 8. 风险与兜底

| 风险 | 影响 | 兜底 |
|---|---|---|
| Astra 相机被占用 | 无法读取图像 | 停掉旧相机节点，只在 `icar_ros2` 容器内启动 |
| YOLO 环境不兼容 | 无法推理 | 使用 OpenCV 兜底检测，保留 YOLO 接口和选型说明 |
| 公开数据集视角不匹配 | 检测效果差 | 用小车低机位补采数据，少量微调 |
| 小车联调时间不足 | 中期演示失败 | 使用 `fake_camera` 无车演示完整 ROS2 话题链路 |
| 视觉控制与导航冲突 | 小车运动不安全 | 视觉只发布速度建议，不直接发 `/cmd_vel` |
