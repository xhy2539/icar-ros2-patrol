# 接口设计文档

> **icar-ros2-patrol** 模块间接口设计说明（IF-01 ~ IF-10）

## IF-01：APP/网页 → 底盘控制

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/cmd_vel` |
| **消息类型** | `geometry_msgs/msg/Twist` |
| **方向** | `app_control_node` → 底盘控制模块 |
| **触发条件** | APP 端用户操作（虚拟摇杆/按钮） |
| **数据内容** | `linear.x` 前进/后退速度, `angular.z` 转向角速度 |
| **备注** | 避障模块和任务调度模块可在紧急情况下 override 此 Topic |

### 控制指令映射

| APP 操作 | linear.x (m/s) | angular.z (rad/s) |
|----------|---------------|-------------------|
| 前进 | +0.3 ~ +0.5 | 0.0 |
| 后退 | -0.2 ~ -0.3 | 0.0 |
| 左转 | 0.0 | +0.5 ~ +1.0 |
| 右转 | 0.0 | -0.5 ~ -1.0 |
| 停止 | 0.0 | 0.0 |

## IF-02：雷达模块 → 避障模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/scan` |
| **消息类型** | `sensor_msgs/msg/LaserScan` |
| **方向** | `lidar_node` → `obstacle_avoid_node` |
| **触发条件** | 激光雷达持续扫描（10-20Hz） |
| **数据内容** | `ranges[]` 距离数组, `angle_min/max/increment`, `range_min/max` |
| **备注** | 思岚A1 360° 激光雷达 |

### 避障判定规则

| 条件 | risk_level | 动作 |
|------|-----------|------|
| min_distance > 1.0m | safe | none |
| 0.5m < min_distance ≤ 1.0m | warning | slow_down |
| 0.3m < min_distance ≤ 0.5m | danger | stop |
| min_distance ≤ 0.3m | danger | stop（紧急停止） |

## IF-03：避障模块 → 任务调度/底盘

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/obstacle_status`（状态）, `/cmd_vel`（停止/避让指令） |
| **消息类型** | 自定义 `ObstacleStatus.msg` + `geometry_msgs/msg/Twist` |
| **方向** | `obstacle_avoid_node` → `task_manager_node`, 底盘控制模块 |
| **触发条件** | 检测到障碍物距离 < 安全阈值 |
| **数据内容** | `bool is_obstacle`, `float32 min_distance`, `string direction`, `string risk_level`, `string action` |

### 自定义消息定义

```
# ObstacleStatus.msg
bool is_obstacle        # 是否存在障碍物
float32 min_distance    # 最近障碍物距离 (m)
string direction        # 障碍物方位 (front/left/right/back)
string risk_level       # 风险等级 (safe/warning/danger)
string action           # 建议动作 (none/slow_down/stop/turn)
```

## IF-04：SLAM模块 → 导航模块/APP

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/map`（地图数据）, `/pose`（位姿） |
| **消息类型** | `nav_msgs/msg/OccupancyGrid` + `geometry_msgs/msg/PoseStamped` |
| **方向** | `slam_node` → `navigation_node`, `app_control_node` |
| **触发条件** | 建图更新 / 定位更新 |
| **数据内容** | 栅格地图 + 地图元信息（分辨率/原点/宽高） + 机器人实时位姿 |
| **备注** | 地图可保存为 .pgm + .yaml 文件 |

## IF-05：APP/任务调度 → 导航模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/goal_pose` |
| **消息类型** | `geometry_msgs/msg/PoseStamped` |
| **方向** | `app_control_node` / `task_manager_node` → `navigation_node` |
| **触发条件** | 用户点击目标点 或 任务调度下发巡检点 |
| **数据内容** | `pose.position`（x, y, z）, `pose.orientation`（四元数）, `header.frame_id`="map" |
| **备注** | 巡检路线：A(1,0,0) → B(2,1,0) → C(3,0,0)，按顺序下发 |

## IF-06：视觉模块 → 任务调度/APP/LLM

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/vision/detections` |
| **消息类型** | 自定义 `DetectionArray.msg` |
| **方向** | `vision_node` → `task_manager_node`, `app_control_node`, `llm_gateway_node`(P2) |
| **触发条件** | 每次检测帧输出 (10-30Hz) |
| **数据内容** | 检测目标列表（类别/置信度/边界框）+ 截图路径 + 时间戳 |
| **备注** | 视觉检测结果作为巡检报告的素材 |

### 自定义消息定义

```
# Detection.msg
string class_name        # 目标类别 (person/obstacle/sign/unknown)
float32 confidence       # 置信度 0.0-1.0
int32 x_min              # 边界框左上 x
int32 y_min              # 边界框左上 y
int32 x_max              # 边界框右下 x
int32 y_max              # 边界框右下 y
string image_path        # 截图保存路径

# DetectionArray.msg
std_msgs/Header header
Detection[] detections
```

### JSON 输出示例

```json
{
  "module": "vision",
  "event": "object_detected",
  "data": {
    "object": "person",
    "confidence": 0.86,
    "bbox": [120, 80, 300, 420],
    "image_path": "logs/images/point_A_001.jpg"
  }
}
```

## IF-07：传感器模块 → 任务调度/APP/LLM

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/sensor/env_data`（实时数据）, `/sensor/alert`（异常告警） |
| **消息类型** | 自定义 `EnvData.msg` + `SensorAlert.msg` |
| **方向** | `sensor_node` → `task_manager_node`, `app_control_node`, `llm_gateway_node`(P2) |
| **触发条件** | 定时采集 (1Hz) / 异常阈值触发 |
| **数据内容** | 温度/湿度/烟雾/PM2.5/光照/气压 + 时间戳 |

### 自定义消息定义

```
# EnvData.msg
std_msgs/Header header
float32 temperature      # 温度 (℃) - DHT22
float32 humidity         # 湿度 (%) - DHT22
float32 smoke            # 烟雾浓度 (ppm) - MQ-2
float32 pm25             # PM2.5 (μg/m³) - PMS5003
float32 light            # 光照强度 (lux) - BH1750
float32 pressure         # 气压 (hPa) - BMP280

# SensorAlert.msg
string sensor_type       # 传感器类型
float32 current_value    # 当前值
float32 threshold        # 阈值
string severity          # WARN / ERROR
string message           # 告警描述
```

### 异常规则（参考）

| 传感器 | 阈值 | 严重级别 |
|--------|------|----------|
| 温度 > 50℃ | 50.0 | ERROR |
| 烟雾 > 100ppm | 100.0 | ERROR |
| PM2.5 > 150μg/m³ | 150.0 | WARN |
| 湿度 < 20% | 20.0 | WARN |

### JSON 输出示例

```json
{
  "module": "sensor",
  "event": "environment_data",
  "data": {
    "temperature": 28.5,
    "humidity": 62,
    "pm25": 35,
    "smoke": "normal",
    "light": 320,
    "pressure": 1013.2
  }
}
```

## IF-08：任务调度 → APP展示

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/task/status`, `/task/log` |
| **消息类型** | 自定义 `TaskStatus.msg` + `TaskLog.msg` |
| **方向** | `task_manager_node` → `app_control_node` |
| **触发条件** | 任务状态变化 / 巡检事件发生 |
| **数据内容** | 任务ID、状态、步骤进度、日志事件列表 |

### 巡检状态机

```
PENDING → RUNNING → NAVIGATING → CHECKPOINT → DETECTING → COLLECTING
                                                              │
                                                    ┌─────────┴──────────┐
                                                    ▼                    ▼
                                              COMPLETED              FAILED
                                              (正常结束)            (异常终止)
```

## IF-09：任务日志 → LLM 模块（P2 加分项）

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic + Service |
| **Topic 名称** | `/task/log` |
| **Service 名称** | `/llm/parse_task`, `/llm/generate_report` |
| **消息类型** | 自定义 `TaskLog.msg` + `ParseTask.srv` |
| **方向** | `task_manager_node` → `llm_gateway_node` |
| **触发条件** | 用户输入自然语言任务指令 / 巡检任务结束后 |
| **数据内容** | 原始文本指令 + 任务日志 + 导航/检测/传感器结果 |
| **备注** | LLM 模块为后期加分项，接口预留，前期可 mock |

### Service 定义参考

```
# ParseTask.srv
string input_text        # 用户自然语言输入，如"巡检实验室一圈，经过A/B/C点"
---
string task_json         # 解析后的结构化任务 JSON
bool success             # 是否解析成功
string error_msg         # 错误信息
```

### LLM 任务 JSON 示例

```json
{
  "task_type": "patrol",
  "route": ["A", "B", "C"],
  "actions": ["navigate", "avoid_obstacle", "detect_object", "collect_sensor"],
  "safety_rule": "遇到障碍物或烟雾异常时停止并报警"
}
```

## 接口通信总览图

```
┌──────────────┐  IF-01 /cmd_vel     ┌──────────────┐
│  APP 控制台   │ ──────────────────▶│   底盘控制    │
│ (app_control) │                    │   (AT32)     │
└──┬───┬───┬───┘                    └──────────────┘
   │   │   │                              ▲
   │   │   │ IF-04 /map, /pose            │ IF-03 /cmd_vel
   │   │   ▼                              │ (emergency stop)
   │   │ ┌──────────┐  IF-02 /scan  ┌──────────────┐
   │   │ │ SLAM/导航 │ ◀─────────────│  激光雷达    │
   │   │ │ (slam/   │               │  (lidar)     │
   │   │ │  nav)    │               └──────────────┘
   │   │ └────┬─────┘
   │   │      │ IF-03 /obstacle_status
   │   │      ▼
   │   │ ┌─────────────────────────────────────────┐
   │   │ │        任务调度 (task_manager)           │
   │   │ │  ◀── IF-06 /vision/detections           │
   │   │ │  ◀── IF-07 /sensor/env_data, /alert     │
   │   │ │  ──▶ IF-08 /task/status, /task/log      │
   │   │ └────────────────┬────────────────────────┘
   │   │                  │ IF-09 /task/log (P2)
   │   │                  ▼
   │   │           ┌──────────────┐
   │   │           │  LLM 网关     │
   │   └───────────│ (llm_gateway) │
   │     IF-09     └──────────────┘
   │  /llm/parse_task
   │
   └─── IF-06, IF-07 /vision/detections, /sensor/env_data (APP展示)
```
