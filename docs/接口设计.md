# 接口设计文档

> **icar-ros2-patrol** 模块间接口设计说明

---

## 1. APP控制模块 → 小车底盘控制模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/cmd_vel` |
| **消息类型** | `geometry_msgs/msg/Twist` |
| **方向** | `app_control_node` → 底盘驱动节点 |
| **触发条件** | APP 端用户操作（虚拟摇杆/按钮） |
| **数据内容** | `linear.x` 前进/后退速度, `angular.z` 转向角速度 |
| **备注** | 避障模块可 override 此 Topic（发布停止指令） |

### 接口示例

```python
# app_control_node 发布运动指令
twist = Twist()
twist.linear.x = 0.5   # 前进 0.5 m/s
twist.angular.z = 0.0  # 不转向
publisher.publish(twist)
```

## 2. 雷达避障模块 → 任务调度模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/obstacle_status` |
| **消息类型** | 自定义 `ObstacleStatus.msg` |
| **方向** | `obstacle_avoid_node` → `task_manager_node` |
| **触发条件** | 检测到障碍物距离 < 安全阈值 |
| **数据内容** | `bool is_obstacle`, `float32 min_distance`, `string direction` |
| **备注** | 任务调度模块收到后决策是否暂停/取消当前任务 |

### 自定义消息定义参考

```
# ObstacleStatus.msg
bool is_obstacle       # 是否存在障碍物
float32 min_distance   # 最近障碍物距离 (m)
string direction       # 障碍物方位 (front/left/right/back)
```

## 3. SLAM导航模块 → APP展示模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic + Service |
| **Topic 名称** | `/map`（地图数据）, `/nav_status`（导航状态） |
| **消息类型** | `nav_msgs/msg/OccupancyGrid` + 自定义 `NavStatus.msg` |
| **方向** | `slam_node` / `navigation_node` → `app_control_node` |
| **触发条件** | 建图更新 / 导航状态变化 |
| **数据内容** | 栅格地图 + 当前位姿 + 目标位姿 + 路径规划状态 |
| **备注** | APP 端以可视化方式展示地图和导航进度 |

### 导航状态消息定义参考

```
# NavStatus.msg
string status           # IDLE / NAVIGATING / ARRIVED / FAILED
float32 progress        # 导航进度百分比 0.0-1.0
float32 distance_remain # 剩余距离 (m)
string message          # 状态描述
```

## 4. 视觉检测模块 → 任务调度模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/vision/detections` |
| **消息类型** | 自定义 `DetectionArray.msg` |
| **方向** | `vision_node` → `task_manager_node` |
| **触发条件** | 每次检测帧输出 |
| **数据内容** | 检测目标列表（类别/置信度/边界框） + 时间戳 |
| **备注** | 任务调度模块根据检测结果触发巡检记录 |

### 检测结果消息定义参考

```
# Detection.msg
string class_name       # 目标类别
float32 confidence      # 置信度 0.0-1.0
int32 x_min             # 边界框
int32 y_min
int32 x_max
int32 y_max

# DetectionArray.msg
std_msgs/Header header
Detection[] detections
```

## 5. 传感器模块 → 任务调度模块

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Topic |
| **Topic 名称** | `/sensor/env_data` |
| **消息类型** | 自定义 `EnvData.msg` |
| **方向** | `sensor_node` → `task_manager_node` |
| **触发条件** | 定时采集（如每秒1次）或阈值触发 |
| **数据内容** | 温度/湿度/烟雾浓度/PM2.5/光照/气压 + 时间戳 |
| **备注** | 任务调度模块校验异常规则，触发告警记录 |

### 环境数据消息定义参考

```
# EnvData.msg
std_msgs/Header header
float32 temperature     # 温度 (℃)
float32 humidity        # 湿度 (%)
float32 smoke           # 烟雾浓度 (ppm)
float32 pm25            # PM2.5 (μg/m³)
float32 light           # 光照强度 (lux)
float32 pressure        # 气压 (hPa)
```

## 6. 任务日志 → LLM 模块（P2 加分项）

| 项目 | 说明 |
|------|------|
| **通信方式** | ROS2 Service / HTTP REST（待定） |
| **Service 名称** | `/llm/parse_task` |
| **消息类型** | 自定义 `ParseTask.srv` |
| **方向** | `task_manager_node` → `llm_gateway_node` |
| **触发条件** | 用户输入自然语言任务指令 |
| **数据内容** | 原始文本指令 + 结构化任务 JSON |
| **备注** | LLM 模块为后期加分项，接口预留，前期可 mock |

### Service 定义参考

```
# ParseTask.srv
string input_text       # 用户自然语言输入
---
string task_json        # 解析后的结构化任务 JSON
bool success            # 是否解析成功
string error_msg        # 错误信息
```
