# iCar 智能巡检小车 — 中期答辩汇报

> 日期：2026.07.11

## 一、项目概况

基于 Jetson Orin Nano 的 ROS2 智能巡检小车，实现底盘控制、SLAM 建图、视觉检测、APP 控制、云平台远程通信，后续接入 LLM 语音交互和巡检闭环。

- 平台：Jetson Orin Nano + ROS2 Foxy
- 硬件：思岚A1激光雷达 / 奥比中光深度相机 / AT32底盘
- 开发策略：Mock 先行、统一 Topic 接口、松耦合并行开发

## 三、拟采用的解决方案与进度

### 3.1 架构方案：Mock 先行

所有模块通过统一的 ROS2 Topic 接口通信，task_manager 不感知对面是 mock 还是真实模块。各成员独立开发、独立测试，互不阻塞。

**进度**：✅ 架构层完成。11 个节点，9 msg + 2 srv，mock 闭环通过，真实模块逐步替换。

### 3.2 底盘控制

- 串口直控：`Rosmaster_Lib` → `/dev/myserial` → AT32 底盘
- ROS2 控制：`Mcnamu_driver_X3` 订阅 `/cmd_vel`，键盘节点(m3)发布速度指令
- APP 控制：Flutter APP → WebSocket → app.py → 底盘

**进度**：✅ 三种方式均在车上验证通过。SLAM 建图完成（gmapping + 保存地图）。

### 3.3 导航

激光雷达数据接入 ROS2 `/scan`，gmapping 实时建图，Nav2 全局规划 + 局部避障，发布 `/nav_status` 给 task_manager。

**进度**：🔄 代码完成（5节点），SLAM 建图在车上验证通过，自主导航 + 避障待真车联调。

### 3.4 视觉

深度相机采集图像，HSV 颜色检测 + 可选 YOLO 后端。模拟摄像头节点支持无硬件开发。目标跟踪输出速度建议。

**进度**：🔄 代码完成（5节点），模拟摄像头可用，真摄像头上车待测。

### 3.5 本地 LLM 部署（熊浩宇）

Mac M5 本地运行 llama.cpp-omni（MiniCPM-o 4.5，Q4_K_M 量化，7.3GB），Metal 加速。CLI 模式 LLM+TTS 语音交互验证通过。后续对接车端 llm_gateway_node 实现语音指令→任务执行→报告生成。

**进度**：🔄 模型加载和语音生成验证成功，HTTP API Server 待修复。

### 3.6 LLM 网关（王璐）

llm_gateway_node 提供自然语言→TaskRequest 解析和巡检日志→报告生成。有网时调 LLM API，无网时模板兜底。

**进度**：✅ 模板兜底可用，ParseTask + GenerateReport 服务接口已定义。

### 3.7 云平台通信

双模通信架构：局域网内 APP 直连小车 WebSocket（低延迟视频流+控制）；远程时通过腾讯云 MQTT Broker 中转（手机 4G/小车 NBIOT）。cloud_bridge 节点在车上做 ROS2↔MQTT 桥接。

**进度**：✅ 腾讯云 MQTT + WebSocket 桥接已部署 24h 运行。cloud_bridge 节点代码完成，待车上部署。

### 3.8 Flutter APP

5 页面（控制台/状态/传感器/日志/设置），WebSocket 直连小车，长按走松手停，支持视频流播放。

**进度**：✅ 基础控制可用。多指令并行、云平台对接待完成。
