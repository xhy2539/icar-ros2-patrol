> 云服务器已搭建完成，小车可通过 NBIOT/4G 连云，手机 APP 可在任何地方远程控制，不再需要同一局域网。

|------|------|------|
| MQTT | `82.156.132.43:1883` (用户:icar 密码:icar123456) | 消息中转 |
| WebSocket | `ws://82.156.132.43:5000/ws` | APP 连接入口 |
| API | `http://82.156.132.43:5000/` | 健康检查 |

**服务器配置**：见 `docs/云服务器配置`（含密码，已 gitignore，组成员私聊拿）。

## 二、架构变化

```
之前（局域网）:
  手机 ──WebSocket──▶ 小车IP:6500        ← 必须同一个WiFi

现在（云平台）:
  手机 ──WebSocket──▶ 82.156.132.43:5000 ──MQTT──▶ 小车 cloud_bridge_node
   4G/5G/WiFi         腾讯云服务器                 NBIOT/4G/WiFi
   (任何网络)                                      (任何网络)
```

## 四、Topic 映射（车 ↔ 云）

| ROS2 Topic | MQTT Topic | 方向 | 用途 |
|------------|-----------|------|------|
| `/task/request` | `/icar/cmd` | 手机→车 | 巡检指令 |
| `/task/status` | `/icar/status` | 车→手机 | 巡检进度 |
| `/sensor/alert` | `/icar/alert` | 车→手机 | 异常告警 |
| `/task/log` | `/icar/log` | 车→手机 | 任务日志 |

> 如需新增 Topic，在 `cloud_bridge_node.py` 的 `__init__` 里加一行 `create_subscription` 即可。

## 五、通信模式

```
正常巡检（低带宽）:
  手机发指令 → MQTT /icar/cmd → 小车开始巡逻
  小车每步回报 /icar/status → 手机显示进度
  异常时推送 /icar/alert → 手机弹窗

视频流（高带宽）:
  局域网内直接看 MJPEG: http://小车IP:6500/video_feed
  远程看：后续加 WebRTC 或截图推送（NBIOT 带宽不够传视频）

LLM 语音（后续）:
  手机录音 → 云服务器做 ASR → 转结构化任务 → MQTT → 小车执行
  小车执行结果 → 云服务器做报告生成 → 推回手机
```

## 六、测试方法

```bash
# 1. 测试云服务器连通
curl http://82.156.132.43:5000/

# 2. 用 mosquitto 模拟手机发指令
mosquitto_pub -h 82.156.132.43 -u icar -P icar123456 \
  -t /icar/cmd -m '{"action":"patrol","route":["A","B","C"]}'

# 3. 监听小车发来的数据
mosquitto_sub -h 82.156.132.43 -u icar -P icar123456 -t /icar/#

# 4. 小车端启动 cloud_bridge
docker exec icar_ros2 bash -c "
  source /opt/ros/foxy/setup.bash
  source /root/ros2_ws/install/setup.bash
  ros2 run cloud_bridge cloud_bridge_node
"
```
