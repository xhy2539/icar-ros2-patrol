```

```

## 四、Topic 映射（车 ↔ 云）

| ROS2 Topic | MQTT Topic | 方向 | 用途 |
|------------|-----------|------|------|
| `/task/request` | `/icar/cmd` | 手机→车 | 巡检指令 |
| `/cmd_vel_cloud` | `/icar/control` | 手机→车 | 带短租约的远程方向控制 |
| `/task/status` | `/icar/status` | 车→手机 | 巡检进度 |
| `/nav_status` | `/icar/nav` | 车→手机 | 低频导航状态 |
| `/obstacle_status` | `/icar/obstacle` | 车→手机 | 低频避障状态，危险事件立即发送 |
| `/sensor/env_data` | `/icar/env` | 车→手机 | 环境传感器数据 |
| `/sensor/alert` | `/icar/alert` | 车→手机 | 异常告警 |
| `/task/log` | `/icar/log` | 车→手机 | 任务日志 |
| - | `/icar/online` | 车→手机 | retained 在线状态 / Last Will |
| - | `/icar/ack` | 车→手机 | 巡检任务接受或拒绝结果 |
| loopback JPEG | `/icar/snapshot/request` | 手机→车 | 请求原始或标注截图 |
| loopback JPEG | `/icar/snapshot` | 车→手机 | Base64 JPEG 或明确错误 |

> 如需新增 Topic，在 `cloud_bridge_node.py` 的 `__init__` 里加一行 `create_subscription` 即可。

## 五、通信模式

```
正常巡检（低带宽）:
  手机发指令 → MQTT /icar/cmd → 小车开始巡逻
  小车每步回报 /icar/status → 手机显示进度
  异常时推送 /icar/alert → 手机弹窗

视频流（高带宽）:
  局域网内直接看 MJPEG: http://小车IP:6500/video_feed
  远程截图：APP 经 MQTT 按需请求，车端从 127.0.0.1:6502 取一帧
  远程连续视频：后续使用 WebRTC；NBIOT 不承载连续视频

LLM 语音（后续）:
  手机录音 → 云服务器做 ASR → 转结构化任务 → MQTT → 小车执行
  小车执行结果 → 云服务器做报告生成 → 推回手机
```

## 六、测试方法

```bash
# 1. 测试云服务器连通
curl http://82.156.132.43:5000/

# 2. 用 mosquitto 模拟手机发指令
mosquitto_pub -h 82.156.132.43 -u icar -P icar123456 \
  -t /icar/cmd -m '{"action":"patrol","route":["A","B","C"]}'

# 3. 监听小车发来的数据
mosquitto_sub -h 82.156.132.43 -u icar -P icar123456 -t /icar/#

# 4. 请求一张原始画面（device_id 为空时）
NOW=$(date +%s)
mosquitto_pub -h 82.156.132.43 -u icar -P icar123456 \
  -t /icar/snapshot/request \
  -m "{\"request_id\":\"manual-$NOW\",\"annotated\":false,\"issued_at\":$NOW,\"expires_at\":$((NOW+10))}"

# 5. 小车端启动 cloud_bridge
docker exec icar_ros2 bash -c "
  source /opt/ros/foxy/setup.bash
  source /root/ros2_ws/install/setup.bash
  ros2 run cloud_bridge cloud_bridge_node
"
```

## 七、版本化部署与回滚

不让公网 CI 主动 SSH 进入 NAT/4G 后的小车，改为 GitHub 构建、小车拉取指定版本：

1. `.github/workflows/release.yml` 在版本 Tag 或人工触发后运行全部离线测试。
2. 流水线生成 APK、ROS2/源码包及各自的 SHA-256。
3. `pull_release.sh` 只接受明确指定的 HTTPS 发布地址，不自动跟随 `latest`。
4. `install_release.sh` 校验后解压到 `/home/jetson/icar-deploy/releases/<version>`。
5. 健康检查通过才保留新 `current`；失败会自动恢复旧版本。
6. `rollback_release.sh` 可手动切回 `previous`。

首次在小车启用时需要安装发布包内的两个 systemd 服务文件。此步骤和 ROS2 编译必须等小车恢复后执行；构建、校验、暂存和回滚逻辑均已在电脑隔离目录演练通过。
