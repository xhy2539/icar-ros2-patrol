# 模块集成产出清单

> 每个组员接入前确认 Topic 名称和消息字段与下表一致。格式对齐后 `task_manager_node` 不用改代码。

---

## 李雨晨 — APP/网页控制台

**必须发布：**

| Topic | 类型 | 说明 |
|-------|------|------|
| `/task/request` | `icar_interfaces/TaskRequest` | 点击"开始巡检"时发布 |
| `/cmd_vel` | `geometry_msgs/Twist` | 手动控制，必须有停止按钮 |

**TaskRequest 示例：**

```
task_type = "patrol"
route = ["A", "B", "C"]
params = '{"stop_on_obstacle": true}'
```

**必须订阅展示：**

| Topic | 展示内容 |
|-------|----------|
| `/task/status` | 任务状态、当前步骤、进度 |
| `/task/log` | 任务日志列表 |
| `/sensor/env_data` | 温湿度/烟雾/PM2.5/光照/气压 |
| `/vision/detections` | 检测目标类别/置信度/截图 |
| `/nav_status` | 导航状态 |

**验收产出：** 控制界面截图、手动控制视频、状态面板截图。

## 曹莹 — 雷达导航模块

**必须发布：**

| Topic | 类型 | 说明 |
|-------|------|------|
| `/scan` | `sensor_msgs/LaserScan` | 思岚A1 原始扫描（已有） |
| `/odom` | `nav_msgs/Odometry` | 底盘里程计（已有） |
| `/map` | `nav_msgs/OccupancyGrid` | SLAM 栅格地图 |
| `/nav_status` | `icar_interfaces/NavStatus` | 导航状态，必须严格匹配字段 |
| `/obstacle_status` | `icar_interfaces/ObstacleStatus` | 避障状态 |

**当前进展说明：**

- `/scan` 已接入真实雷达源，当前不再由项目内 mock 生成
- `/nav_status`、`/obstacle_status` 已切到正式消息类型
- `/obstacle_status` 默认由真实 `/scan` 前方扇区判定生成；`/map`、`/pose`、`/nav_status` 的内部生成逻辑仍处于 mock 过渡态
- 下一阶段优先结合手册 `3.7` 持续把 `SLAM -> 导航` 的内部数据源切到真实链路，并补齐真机避障视频/日志证据

**NavStatus 字段：**

```
string status           # IDLE / NAVIGATING / ARRIVED / FAILED
float32 progress        # 0.0 ~ 1.0
float32 distance_remain # 剩余距离（米）
string message          # "正在前往B点"
```

**ObstacleStatus 字段：**

```
bool is_obstacle        # 是否有障碍物
float32 min_distance    # 最近距离（米）
string direction        # front / left / right / back
string risk_level       # safe / warning / danger
string action           # none / slow_down / stop / turn
```

> task_manager 收到 `risk_level == "danger"` 时自动紧急停止。当前接口已对齐，后续切真时不应再修改消息字段。

**验收产出：** 雷达截图、RViz 建图截图、`.pgm+.yaml` 地图文件、目标点导航视频、避障视频。

> 备注：截至当前阶段，仅 `/scan` 的真实接入可以进入取证；其余产出仍待真数据链路继续替换后补齐。

## 韦雪 — 视觉模块

**必须发布：**

| Topic | 类型 | 频率 |
|-------|------|------|
| `/vision/detections` | `icar_interfaces/DetectionArray` | 每帧/每次检测 |

**Detection 字段：**

```
string class_name    # person / obstacle / sign / unknown
float32 confidence   # 0.0 ~ 1.0
int32 x_min y_min    # bbox 左上
int32 x_max y_max    # bbox 右下
string image_path    # "logs/images/point_A_001.jpg"
```

**验收产出：** 摄像头画面截图、检测框截图或视频。

## 王璐 — 传感器模块

**必须发布：**

| Topic | 类型 | 频率 |
|-------|------|------|
| `/sensor/env_data` | `icar_interfaces/EnvData` | ≥ 1Hz |
| `/sensor/alert` | `icar_interfaces/SensorAlert` | 异常时触发 |

**EnvData 字段：**

```
std_msgs/Header header
float32 temperature   # ℃
float32 humidity      # %
float32 smoke         # ppm
float32 pm25          # μg/m³
float32 light         # lux
float32 pressure      # hPa
```

**异常阈值：**

| 传感器 | 阈值 | 级别 |
|--------|------|------|
| 温度 > 50℃ | 50.0 | ERROR |
| 烟雾 > 100ppm | 100.0 | ERROR |
| PM2.5 > 150μg/m³ | 150.0 | WARN |
| 湿度 < 20% | 20.0 | WARN |

**SensorAlert 字段：**

```
string sensor_type    # smoke / temperature / pm25 / humidity
float32 current_value
float32 threshold
string severity       # WARN / ERROR
string message        # "烟雾浓度超标"
```

> task_manager 收到 `severity == "ERROR"` 时自动紧急停止。

**验收产出：** 至少一种真实传感器数据截图、异常告警记录。

## 熊浩宇 — LLM/报告（P2 加分）

**必须提供：**

| 能力 | 输入 | 输出 |
|------|------|------|
| 任务解析 | 自然语言指令 | 结构化任务 JSON |
| 异常解释 | 传感器/视觉/避障异常 | 文字解释和建议 |
| 报告生成 | `/task/log` 日志列表 | 巡检报告 |

**安全边界：** LLM 不发布 `/cmd_vel`，只输出 JSON 或文本，任务 JSON 必须经 task_manager 白名单校验。

## 集成替换顺序

```
Step 1: python3 scripts/run_mock_demo.py       ← 离线验证
Step 2: ./scripts/start_mock_demo.sh            ← ROS2 mock 闭环
Step 3: 曹莹替换真实导航/SLAM/避障               ← 小车能动了
Step 4: 王璐替换真实传感器                        ← 有环境数据
Step 5: 韦雪替换真实视觉                          ← 能检测目标
Step 6: 李雨晨替换真实 APP                        ← 完整巡检闭环
Step 7: 接入 LLM 任务解析/语音/报告                ← P2 加分
```

> 只要 Topic 名和字段不变，task_manager_node 不用改。

## 当前 mock 验证环境

| 组件 | 节点 | 作用 |
|------|------|------|
| 任务调度 | `task_manager_node` | 状态机 + 日志 + 安全停止 |
| 模拟 APP | `mock_app_node` | 自动下发 A/B/C 巡检任务 |
| 模拟导航 | `mock_navigation_node` | 模拟到点，发布 `/nav_status` |
| 模拟传感器 | `mock_sensor_node` | 发布 `/sensor/env_data` |
| 模拟视觉 | `mock_vision_node` | 发布 `/vision/detections` |
| 报告生成 | `report_generator_node` | 订阅 `/task/log` 打印巡检报告 |

**启动 mock 全链路：**

```bash
# ROS2 版
source /opt/ros/foxy/setup.bash && source install/setup.bash
./scripts/start_mock_demo.sh

# 离线版（不需要 ROS2）
python3 scripts/run_mock_demo.py
```
