# 系统工作模式说明

## 一句话理解

用户通过 APP 下发巡检任务 → task_manager 调度各模块 → 小车依次前往 A/B/C 点 → 每到一个点拍照+采集传感器 → 生成巡检报告。

---

## 一次巡检的数据流

```
时间线 ──────────────────────────────────────────────────────▶

用户             task_manager          导航         视觉       传感器
 │                    │                  │           │          │
 │─"开始巡检A,B,C"──▶│                  │           │          │
 │                    │                  │           │          │
 │                    │ PENDING→RUNNING  │           │          │
 │                    │                  │           │          │
 │                    │ RUNNING→NAVIGATING│          │          │
 │                    │──目标点A───────▶│           │          │
 │                    │                  │─前往A...─│          │
 │                    │                  │          │          │
 │                    │◀───ARRIVED──────│           │          │
 │                    │                  │           │          │
 │                    │ NAVIGATING→CHECKPOINT       │          │
 │                    │ 📝 打卡：时间/坐标/A点       │          │
 │                    │                  │           │          │
 │                    │ CHECKPOINT→DETECTING        │          │
 │                    │                  │──拍照────▶│          │
 │                    │◀──检测到person───┼──────────│          │
 │                    │ 📝 记录检测结果   │           │          │
 │                    │                  │           │          │
 │                    │ DETECTING→COLLECTING        │          │
 │                    │                  │           │──读数──▶│
 │                    │◀────温度28/湿度62────────────┼─────────│
 │                    │ 📝 记录传感器数据  │           │          │
 │                    │                  │           │          │
 │                    │ 还有B、C → 回到NAVIGATING    │          │

```

## 两种控制模式

```
模式1: 手动控制（P0 基础）
─────────────────────────
  用户按"前进"按钮
       │
       ▼
  app_control_node ──/cmd_vel──▶ 底盘 ──▶ 小车前进
       │
  用户松手/按"停止"
       │
       ▼
  /cmd_vel = (0, 0) ──▶ 小车停止

  特点：直接控制，不经过 task_manager


模式2: 自动巡检（P1 闭环）
─────────────────────────
  用户点"开始巡检"或说"去ABC点巡检"
       │
       ▼
  /task/request ──▶ task_manager ──▶ 状态机自动调度
       │                                │
       │                    ┌───────────┼───────────┐
       │                    ▼           ▼           ▼
       │                 导航去A    拍照检测    采集传感器
       │                    │           │           │
       │                    └───────────┼───────────┘
       │                                ▼
       │                          task_manager
       │                          判断下一步
       │                          或紧急停止

  特点：全自动，task_manager 是唯一决策者
```

## 安全机制

```
             obstacle_avoid_node ──────────┐
             (雷达前方0.3m有障碍)           │
                                           ▼
    sensor_node ────────┐          task_manager_node
    (烟雾>100ppm,ERROR)  │          │
                        ├─────────▶│ _emergency_stop()
    vision_node ────────┘          │
    (不重要，仅供参考)              ▼
                            发布 /cmd_vel = (0,0)
                                   │
                                   ▼
                              底盘立即停止
```

> 三条路径可以触发紧急停止，但**只有 task_manager 和 obstacle_avoid 能发停止指令**。LLM 绝对不能控制底盘。

## 开发策略：Mock 先行

```
现在 (7.7)                      7.10                          7.14
   │                              │                              │
   │ mock闭环 ✅                   │ 真实模块替换                 │ 答辩
   │                              │                              │
   ├─ mock_app ──────────┐       ├─ 真APP替换mock_app           │
   ├─ mock_navigation ───┤       ├─ 真导航替换mock_nav          │
   ├─ mock_sensor ───────┼──▶   ├─ 真传感器替换mock_sensor     │
   ├─ mock_vision ───────┘       ├─ 真视觉替换mock_vision       │
   │                              │                              │
   └─ task_manager (真实，不改) ──┴─ task_manager (不变！) ─────┘
```

关键设计：mock 节点和真实节点发**同样的 Topic、同样的字段**，task_manager 分不清对面是 mock 还是真模块。所以它从一开始就是"真实可用"的，组员接入时不需要你改代码。
