# 直接语音运动设计

## 目标

用户说出受支持的移动指令后，小车直接执行，不再要求二次确认。

## 规则

- 豆包对受支持的短时、低速运动请求只输出 `执行任务：<指令>`。
- 语音路由仅把带有该前缀的回复发送给 LLM 工具路由。
- 普通聊天或说明文本不会进入动作路由，避免其中的“前进”等词触发动作。
- 运动仍由现有 `move_robot` 工具限制在低速、短时范围，并经过 velocity_mux、避障和急停。

## 验证

- 非执行前缀的回复不发布 `/llm/user_command`。
- `执行任务：前进一秒` 发布对应的 LLM 指令。
- 语音节点的系统提示词包含直接执行的固定输出要求。
