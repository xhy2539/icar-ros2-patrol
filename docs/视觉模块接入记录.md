# 视觉模块接入记录

## 当前统一的小车使用规则

不要手工修改 `/dev/rplidar`、`/dev/myserial`、`/dev/sensors`、
`/dev/video0`、`ttyUSB*` 等设备映射。

导航/建图使用：

```bash
source ~/.bashrc
s
d
```

进入 `autodrive_ros2` 容器后，再执行 `m1/m2/m3/m4/n1/n2...`。

视觉/icar 使用：

```bash
source ~/.bashrc
is
id
```

进入 `icar_ros2` 容器后，再执行 `m1/m2/m3/n1/n2/n3/n4/n5`。

不要在宿主机直接执行 `m1/n1/n2`，也不要使用
`docker exec $(docker ps -q)` 这种不确定容器的方式。

## 上车后的相机接入检查

进入 `icar_ros2` 后：

```bash
source /opt/ros/foxy/setup.bash
m1
ros2 topic list | grep -E "image|camera|depth|rgb|color|astra|points"
```

记录真实 RGB 图像 topic。常见候选：

```text
/camera/color/image_raw
/camera/depth/image_raw
```

确认图像是否发布：

```bash
ros2 topic info /camera/color/image_raw
ros2 topic hz /camera/color/image_raw
```

## 本仓库预开发入口

本地已准备 ROS2 包：

```text
vision/vision_patrol
```

构建：

```bash
colcon build --symlink-install --packages-select vision_patrol
source install/setup.bash
```

相机接入测试：

```bash
ros2 run vision_patrol camera_probe --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_first_frame:=true
```

视觉管线占位节点：

```bash
ros2 run vision_patrol vision_node --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p mode:=detect
```

后续 YOLO 目标识别和道路检测逻辑都接入
`vision_patrol/vision_node.py`。

## 不上车的仿真测试

在 Ubuntu 20.04 + ROS2 Foxy 虚拟机中测试：

```bash
cd ~/ros2_ws
colcon build --symlink-install --packages-select vision_patrol
source install/setup.bash
```

终端 1：发布仿真相机。

```bash
ros2 run vision_patrol fake_camera --ros-args \
  -p image_topic:=/camera/color/image_raw
```

终端 2：验证相机帧。

```bash
ros2 run vision_patrol camera_probe --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_first_frame:=true
```

终端 3：运行视觉节点。

```bash
ros2 run vision_patrol vision_node --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p enable_road_detection:=true \
  -p publish_annotated:=true
```

终端 4：查看输出。

```bash
ros2 topic echo /vision/detections
ros2 topic list | grep vision
```

仿真节点会生成路面、车道线、红色路障、绿色标志物和黄色行人，用于先跑通
`/camera/color/image_raw -> vision_node -> /vision/detections` 链路。

## 截图/数据集采集策略

不要默认保存每一帧。摄像头 15-30 FPS 连续保存会很快占满磁盘，也会影响小车
推理性能。视觉模块采用可控采集：

- 默认不保存图片。
- App 或任务调度可发布 `/vision/capture_command` 触发单张截图。
- 需要采集训练集时，可设置每 N 秒保存一张。
- 使用 `max_images` 限制最多保存张数。

启动采集节点：

```bash
ros2 run vision_patrol dataset_recorder --ros-args \
  -p image_topic:=/camera/color/image_raw \
  -p save_dir:=/tmp/icar_vision_dataset \
  -p max_images:=200
```

单张截图命令：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"capture_once\",\"tag\":\"checkpoint_A\"}'}"
```

每 3 秒保存一张：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"set_interval\",\"interval_sec\":3.0}'}"
```

停止定时保存：

```bash
ros2 topic pub --once /vision/capture_command std_msgs/String \
  "{data: '{\"action\":\"stop\"}'}"
```

保存状态发布到 `/vision/capture_status`，供 App 显示保存路径和计数。

## 目标追踪/跟随策略

目标追踪节点订阅 `/vision/detections`，根据 `/vision/target_tracking/command`
选择要追踪的目标类别。节点根据目标框中心位置和框面积估计输出速度建议：

- 目标偏左/偏右：输出 `angular.z`，让车头对准目标。
- 目标框太小：输出正 `linear.x`，表示目标较远，可前进。
- 目标框太大：输出负 `linear.x`，表示目标太近，可后退。
- 目标丢失：发布 0 速度建议。

默认只发布到 `/vision/target_cmd_vel`，不直接控制 `/cmd_vel`。真实上车前需要组内确认
安全策略，由任务调度或底盘控制模块决定是否转发到 `/cmd_vel`。

现在追踪目标由 App/任务命令选择，不默认直接追人。App 可提供按钮：

- 选择追踪目标：人、瓶子、背包、椅子、路障、A/B/C 标志牌。
- 开始追踪。
- 停止追踪。
- 调整最大速度/距离参数。

仿真测试：

```bash
ros2 run vision_patrol target_tracker --ros-args \
  -p detections_topic:=/vision/detections \
  -p cmd_vel_topic:=/vision/target_cmd_vel
```

开始追踪人：

```bash
ros2 topic pub --once /vision/target_tracking/command std_msgs/String \
  "{data: '{\"action\":\"start\",\"class_name\":\"person\"}'}"
```

切换追踪目标：

```bash
ros2 topic pub --once /vision/target_tracking/command std_msgs/String \
  "{data: '{\"action\":\"select_target\",\"class_name\":\"bottle\"}'}"
```

停止追踪：

```bash
ros2 topic pub --once /vision/target_tracking/command std_msgs/String \
  "{data: '{\"action\":\"stop\"}'}"
```

观察：

```bash
ros2 topic echo /vision/target_cmd_vel
ros2 topic echo /vision/target_tracking/status
```

## 目标类别表

当前类别表见 `config/vision_classes.json`。

可直接先用 COCO 预训练 YOLO 的常规类别：

```text
person, backpack, handbag, bottle, chair, stop sign, traffic light
```

需要采集图片并标注/训练的项目自定义类别：

```text
obstacle, sign_A, sign_B, sign_C, water
```

养老院场景中，视觉模块主线先聚焦普通障碍物和积水/湿滑区域；摔倒老人不作为视觉主识别目标，后续按雷达/地图建模或系统融合异常处理。

道路/车道线：

```text
road_lane
```

建议先用 OpenCV 几何检测或分割模型，不强行用普通检测框。
