- `/scan`: `sensor_msgs/msg/LaserScan`

- `/pose`: `geometry_msgs/msg/PoseStamped`
- `/goal_pose`: `geometry_msgs/msg/PoseStamped`
- `/nav_status`: `icar_interfaces/msg/NavStatus`
- `/obstacle_status`: `icar_interfaces/msg/ObstacleStatus`

## 当前已补齐的无车能力

- `task_manager_node` 会根据 `/task/request.route` 查询巡检点配置并发布 `/goal_pose`。
- `navigation_node --mode mock` 继续用 mock 场景演示到点、失败、避障保持。
- `navigation_node --mode real` 不会自动模拟 `ARRIVED`，只保持 `NAVIGATING` 并等待真实导航反馈。
- `nav2_bridge_node --mode real`（新增）：对接 Nav2 NavigateToPose action server，翻译 Nav2 反馈为项目 `NavStatus`。`--mode mock` 行为与 `navigation_node --mode mock` 一致。
- `obstacle_avoid_node` 默认读取真实 `/scan`，无雷达演示时才使用 `--mode mock`。
- `scripts/start_navigation.sh real` 已提供 real-ready 启动壳，保留真实 `slam_toolbox/gmapping` 和 Nav2 接入位置。
- PGM 解析器已支持 P2（ASCII）和 P5（binary）两种格式。真实 SLAM 地图加载时需在 `map_metadata.yaml` 中设置 `invert: true`（标准 ROS PGM 为黑=障碍，与 mock 地图的白=障碍约定相反）。

## 真车可用后只需要补的内容

1. ~~接入真实 `/odom`~~ ✅ 已确认可用（`autodrive_ros2` EKF 节点），无需额外接入。
2. 用 `slam_toolbox` 或 `gmapping` 生成真实 `/map` 和 `/pose`。
3. 保存真实地图 `.pgm + .yaml`，替换当前 mock 地图资源。在 `map_metadata.yaml` 中设置 `invert: true`。
4. 在真实地图上标定 `HOME/A/B/C` 坐标，更新 `config/navigation/mock/checkpoints.yaml` 或迁移到同字段的 real 配置。
5. 启动 Nav2 + `nav2_bridge_node --mode real`（替代 `navigation_node --mode real`）。桥接节点需在 `autodrive_ros2`（domain 30）内运行，与 Nav2 同域。若项目代码未挂载进该容器，需添加目录挂载。
6. 用 RViz 和日志补齐验收证据：雷达截图、建图截图、地图文件、目标点导航视频、避障视频。

---

## 接车验收顺序

1. `ros2 topic echo /scan` 确认雷达数据稳定。  ← ✅ 已确认
2. `ros2 topic echo /odom` 确认里程计稳定。  ← ✅ 已确认，`autodrive_ros2` 内 EKF 节点
3. 启动真实 SLAM，确认 `/map`、`/pose` 可在 RViz 中显示。
4. 保存地图并标定巡检点坐标。
5. 发布单个 `/goal_pose`，确认小车能移动并反馈 `/nav_status`。
6. 下发 `/task/request route=[A,B,C]`，确认 task_manager 会依次发布巡检目标。
7. 放置前方障碍，确认 `/obstacle_status` 到 `danger` 后小车停车。
